# Stock Screener with LINE Notification (株価スクリーナー & LINE通知ツール)

このプロジェクトは、`yfinance`を使用して日米の株価データを取得・テクニカル分析し、あらかじめ設定した条件（シグナル）を満たした銘柄をLINEに通知するPythonスクリプトです。

> [!NOTE]
> LINE Notifyのサービス終了（2025年3月31日）に伴い、本ツールでは **LINE Messaging API** を使用してプッシュ通知を送信します。

---

## 主な機能
1. **テクニカルスクリーニング**:
   - **RSI（相対力指数）**: 売られすぎ（デフォルト: 30以下）、買われすぎ（デフォルト: 70以上）の判定。
   - **移動平均線（MA）クロスオーバー**: 短期移動平均線と長期移動平均線のゴールデンクロス・デッドクロス判定。
   - **前日比急騰・急落**: 前日比の騰落率がしきい値（デフォルト: ±3.0%）を超えた銘柄の検出。
2. **LINEプッシュ通知**: 条件にマッチした銘柄の現在値、前日比、発生したシグナルを分かりやすくフォーマットしてLINEで通知します。
3. **ドライラン（検証用）モード**: `--dry-run` フラグを使うことで、LINE通知を送信せずにターミナル上に通知内容をプレビュー表示できます（LINE APIキーの設定前でもテスト可能）。

## セットアップ手順

### 1. 仮想環境の作成とライブラリのインストール
ターミナルを開き、プロジェクトディレクトリで以下のコマンドを実行します。

```bash
# 仮想環境を作成（推奨）
python -m venv venv

# 仮想環境を有効化 (Windows PowerShell)
.\venv\Scripts\Activate.ps1

# 依存ライブラリのインストール
pip install -r requirements.txt
```

### 2. 設定ファイルの調整 (`config.yaml`)
`config.yaml` をエディタで開き、スクリーニング対象の銘柄や判定しきい値を自由に編集します。
* 日本株は末尾に `.T` を付けます（例: トヨタなら `7203.T`）。
* 米国株はそのままのシンボルを入力します（例: `AAPL`）。

### 3. 環境変数の設定 (`.env`)
`.env.template` ファイルをコピーして、名前を `.env` に変更します。

```bash
copy .env.template .env
```

`.env` ファイルを開き、LINE Developers から取得した認証情報を設定します（取得方法は後述）。

```env
LINE_CHANNEL_ACCESS_TOKEN=取得したチャネルアクセストークン
LINE_USER_ID=取得したあなたのユーザーID
```

## LINE Messaging API の設定手順（通知用）

LINE通知を受け取るためには、無料のLINE公式アカウント（Bot）を作成する必要があります。

1. **LINE Developersへのログイン**:
   - [LINE Developers コンソール](https://developers.line.biz/) に個人のLINEアカウントでログインします。
2. **プロバイダーの作成**:
   - 右上の「コンソール」に入り、「作成」ボタンから新規プロバイダー（例: `MyStockTools`）を作成します。
3. **チャネルの作成**:
   - 作成したプロバイダー内で、「新規チャネル作成」をクリックし、「Messaging API」を選択します。
   - 必要な項目（チャネル名、説明、カテゴリなど）を入力して作成します。
4. **友だち追加（最重要）**:
   - チャネル作成後、「Messaging API設定」タブを開くと、BotのQRコードが表示されます。
   - **ご自身のスマートフォン等でこのQRコードをスキャンし、Botアカウントを友だち追加します。**
5. **チャンネルアクセストークンの発行**:
   - 「Messaging API設定」タブの一番下にある「チャネルアクセストークン（長期）」の「発行」ボタンをクリックし、生成された長いトークンをコピーして、`.env` の `LINE_CHANNEL_ACCESS_TOKEN` に貼り付けます。
6. **ユーザーIDの確認**:
   - 「チャネル基本設定」タブの一番下にある「あなたのユーザーID」（`U`から始まる約33文字の文字列）をコピーし、`.env` の `LINE_USER_ID` に貼り付けます。
   *※LINEアプリ内のIDや表示名ではなく、このコンソールに表示されている開発用IDである必要があります。*

## 実行方法

### ドライラン（コンソール出力テスト）
LINEへの送信は行わず、スクリーニング結果と通知用のメッセージ本文をターミナルに表示します。動作確認の際はまずこちらを実行してください。

```bash
python main.py --dry-run
```

### 本番実行（LINEに通知を送信）
スクリーニング結果をLINEへ送信します。
※条件にマッチする銘柄がない場合は、LINEへの送信は行われずログのみ出力されます。

```bash
python main.py
```

定期実行したい場合は、Windowsの「タスクスケジューラ」や、Linuxの「cron」などにこのスクリプトの実行コマンドを設定してください。
